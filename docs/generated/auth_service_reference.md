## Module Overview

The `auth_service/` module implements the full authentication lifecycle for the GenAI AutoDoc Demo API. It is composed of two sub-modules: `auth.py`, which manages login, logout, token issuance, and token refresh using HMAC-SHA256 signed tokens; and `user.py`, which provides CRUD operations for user accounts with strict input validation and role-based access control. Both modules expose structured exception hierarchies that map directly to HTTP status codes, enabling clean error handling at the FastAPI route layer in `main.py`.

### Functions

#### `login(username: str, password: str) -> TokenPair`

Authenticates the user and returns a fresh `TokenPair`. Increments the failed-attempt counter on failure; resets it on success. Blocks the account after `MAX_LOGIN_ATTEMPTS` consecutive failures (default: 5, configurable via env).

| Parameter | Type | Description |
|---|---|---|
| `username` | `str` | Account's unique identifier |
| `password` | `str` | Plain-text password |

**Returns:** `TokenPair`

**Raises:**
- `InvalidCredentialsError` (401) — wrong username or password
- `AccountLockedError` (403) — account locked after repeated failures
- `InternalAuthError` (500) — unexpected server-side failure

```python
from auth_service.auth import login, InvalidCredentialsError, AccountLockedError

try:
    tokens = login("alice", "secret")
    print(tokens.access_token)
except InvalidCredentialsError:
    print("Bad credentials")
except AccountLockedError:
    print("Account locked — contact support")
```

#### `refresh_access_token(refresh_token: str) -> TokenPair`

Exchanges a valid refresh token for a new `TokenPair`. The old refresh token is revoked atomically (token rotation).

| Parameter | Type | Description |
|---|---|---|
| `refresh_token` | `str` | The current opaque refresh token |

**Returns:** `TokenPair` — new access and refresh tokens

**Raises:**
- `TokenInvalidError` (401) — token not found or already used
- `TokenExpiredError` (401) — refresh session TTL exceeded
- `InternalAuthError` (500) — token rotation failure

```python
from auth_service.auth import refresh_access_token
new_tokens = refresh_access_token(old_refresh_token)
```

### Data Models

#### `UserRecord`
```python
@dataclass
class UserRecord:
    user_id: str           # UUID v4
    username: str          # 3–64 chars: letters, digits, _ or -
    email: str             # Valid RFC-5322 address
    roles: list[str]       # Default: ["viewer"]
    created_at: int        # Unix timestamp
    updated_at: int        # Unix timestamp
    is_active: bool        # False = soft-deleted
```

**Allowed roles:** `viewer`, `developer`, `support`, `ops`, `admin`

### Functions

#### `create_user(username, email, roles) -> UserRecord`

Creates a new user account. Validates all inputs before writing.

| Parameter | Type | Description |
|---|---|---|
| `username` | `str` | Unique display name (3–64 alphanumeric, `_`, `-`) |
| `email` | `str` | Valid RFC-5322 email. Must be unique. |
| `roles` | `list[str]` \| `None` | Role list. Defaults to `["viewer"]` |

**Raises:** `ValidationError` (422), `DuplicateEmailError` (409), `InternalUserError` (500)

```python

```

---

#### `get_user(user_id: str) -> UserRecord`

Retrieves a single user by UUID.

**Raises:** `UserNotFoundError` (404)

---

#### `update_user(user_id, *, email, roles, caller_roles) -> UserRecord`

Updates `email` and/or `roles` on an existing record. Role changes require the caller to hold the `admin` role.

**Raises:** `UserNotFoundError` (404), `ValidationError` (422), `DuplicateEmailError` (409), `PermissionDeniedError` (403), `InternalUserError` (500)

---

#### `delete_user(user_id, *, caller_roles) -> None`

Soft-deletes a user by setting `is_active = False`. Requires `admin` caller role.

**Raises:** `PermissionDeniedError` (403), `UserNotFoundError` (404)

---

#### `list_users(*, active_only: bool = True) -> list[UserRecord]`

Returns all user records sorted by `created_at` ascending. Excludes deactivated accounts when `active_only=True` (default).

## Usage Example

```python
import os
os.environ["JWT_SECRET"] = "dev-secret-do-not-use-in-prod"

from auth_service.auth import login, logout, refresh_access_token, verify_access_token
from auth_service.user import create_user, get_user

# Create a user
user = create_user("alice", "alice@example.com", ["developer"])

# Authenticate
tokens = login("alice", "secret")

# Verify access token
session = verify_access_token(tokens.access_token)
print(session.username, session.roles)

# Refresh
new_tokens = refresh_access_token(tokens.refresh_token)

# Logout
logout(new_tokens.refresh_token)
```
