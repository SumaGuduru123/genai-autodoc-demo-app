## Module Overview

The `frontend-client/` module provides the React-side HTTP client and authentication hooks for the GenAI AutoDoc Demo application. `apiClient.ts` is the base HTTP client that handles bearer-token injection, request timeouts, typed error mapping, and all HTTP verb wrappers. `authHooks.ts` builds on top of the client to provide `useAuth` — a React hook managing the full token lifecycle (login, logout, silent refresh on mount) — and `useRequireRole` for declarative role-based route protection. Together they encapsulate all auth communication so no component ever needs to call the API directly.

### Token Management

#### `setAccessToken(token: string): void`
Stores the access token in memory. Called after a successful login or token refresh.

#### `clearAccessToken(): void`
Clears the in-memory access token. Called on logout. Tokens are never written to `localStorage`.

### Core Request Function

#### `request<T>(method, path, body?, options?): Promise<T>`

The base HTTP function. All convenience wrappers delegate to this.

| Parameter | Type | Description |
|---|---|---|
| `method` | `string` | HTTP verb: `GET`, `POST`, `PUT`, `PATCH`, `DELETE` |
| `path` | `string` | Path relative to `API_BASE_URL` (e.g. `/auth/login`) |
| `body` | `unknown` | Optional JSON-serialisable request payload |
| `options.timeoutMs` | `number` | Per-request timeout override. Default: `10000` ms |
| `options.headers` | `Record<string, string>` | Extra headers merged with defaults |

**Returns:** `Promise<T>` — parsed JSON response body

**Throws:**
- `ApiError` — on 4xx / 5xx responses
- `NetworkTimeoutError` — request aborted after `timeoutMs`
- `NetworkUnavailableError` — network-level failure

```typescript
import { request } from "./apiClient";

const user = await request<User>("GET", "/users/abc-123");
```

### `useAuth(): UseAuthReturn`

Primary authentication hook. Manages the full token lifecycle: login, logout, and silent refresh on mount (reads refresh token from `sessionStorage`).

**Returns `UseAuthReturn`:**

| Field | Type | Description |
|---|---|---|
| `status` | `AuthStatus` | Current auth state |
| `user` | `AuthUser \| null` | Authenticated user |
| `errorCode` | `string \| null` | Error code on failure (e.g. `"INVALID_CREDENTIALS"`) |
| `errorMessage` | `string \| null` | Human-readable error message |
| `login` | `(username, password) => Promise<void>` | Initiates login flow |
| `logout` | `() => Promise<void>` | Revokes token and clears state |
| `hasRole` | `(role: string) => boolean` | Returns `true` if user holds the role |

```tsx
import { useAuth } from "./authHooks";

function App() {
  const { status, user, login, logout, hasRole } = useAuth();

  if (status === "loading") return <Spinner />;
  if (!user) return <LoginForm onSubmit={login} />;

  return (
    <Dashboard
      user={user}
      onLogout={logout}
      isAdmin={hasRole("admin")}
    />
  );
}
```

**Silent refresh on mount:**  
On component mount, `useAuth` checks `sessionStorage` for a stored refresh token. If found, it silently exchanges it for a new token pair and restores the authenticated state without requiring a re-login.

### `useRequireRole(role, user, navigate, redirectTo?): void`

Enforces role-based access inside a component. Redirects to `redirectTo` if the authenticated user does not hold `role`.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `role` | `string` | — | Required role (e.g. `"admin"`) |
| `user` | `AuthUser \| null` | — | Current user from `useAuth` |
| `navigate` | `(path: string) => void` | — | React Router navigate function |
| `redirectTo` | `string` | `"/unauthorized"` | Redirect path on access denial |

```tsx
import { useAuth, useRequireRole } from "./authHooks";
import { useNavigate } from "react-router-dom";

function AdminPage() {
  const { user } = useAuth();
  const navigate = useNavigate();
  useRequireRole("admin", user, navigate);

  return <AdminDashboard />;
}
```

## Usage Example

```tsx
// main.tsx
import React from "react";
import ReactDOM from "react-dom/client";
import { useAuth, useRequireRole } from "./frontend-client/authHooks";
import { get } from "./frontend-client/apiClient";

function ProtectedPage() {
  const { user, logout, hasRole } = useAuth();
  const navigate = (path: string) => window.location.assign(path);

  useRequireRole("developer", user, navigate);

  return (
    <div>
      <p>Welcome, {user?.username}</p>
      {hasRole("admin") && <button>Admin Panel</button>}
      <button onClick={logout}>Logout</button>
    </div>
  );
}
```
